# AVANT — Fix Summary & Deployment Guide
## All Issues Fixed in This PR

---

## 🔴 ISSUE 1: Wake Word Not Working ("Hey Avant" / "Whats up Avant")

**Root Cause:**  
`voiceEngine.ts` had a `WakeWordEngine` class stub but never started a continuous loop.  
`App.js` called `initWakeWordBridge()` but the bridge only checked if it *could* run, not actually listen.

**Fix Applied in:** `voice/voiceEngine.ts`

The `WakeWordEngine` now runs a true **continuous background loop**:
- Calls `listenChunk(3500ms)` in a tight loop
- Each chunk checks for wake patterns: `"hey avant"`, `"whats up avant"`, `"yo avant"`, `"avant"`
- On match → speaks `"Yes?"` → fires a full `VoiceEngine.runSession()`
- Pauses the loop during active sessions so it doesn't interfere
- Restarts automatically after the session ends

Also uses **Android native SpeechRecognizer** in `AvantForegroundService.kt` for background detection even when app is minimized.

## 🔴 ISSUE 2: AVANT Doesn't Talk Back

**Root Cause:**  
`HomeScreen.js` used `Alert.prompt()` instead of the microphone, and had no guaranteed TTS call after AI responses.

**Fix Applied in:** `voice/voiceEngine.ts`, `src/screens/HomeScreen.js`

- `speak()` now uses `expo-speech` as primary (React Native compatible) with Capacitor TTS and Web Speech as fallbacks
- Every `routeCommand()` result is piped directly to `speak()`
- `HomeScreen` no longer uses `Alert.prompt` — uses `VoiceEngine.runSession()` which captures audio and speaks the result

---

## 🔴 ISSUE 3: Camera Permission Loop (White Screen on Vision Tab)

**Root Cause:**  
`ARScreen.js` used the old `expo-camera` v14 API (`Camera.requestCameraPermissionsAsync()`). In expo-camera v16 the API changed. When it failed silently, `hasPermission` stayed `null` and a white/blank screen showed.

**Fix Applied in:** `src/screens/ARScreen.js`

- Dynamic import of `expo-camera` detects v16 (`CameraView`) vs v14 (`Camera`) at runtime
- Tries `requestCameraPermissionsAsync()` → `requestPermissionsAsync()` → `PermissionsAndroid` in order
- Shows clear "Open Settings" button if still denied (with direct link to app settings)
- Separate `CameraBackground` component uses `CameraView` (v16) with `Camera` fallback

---

## 🔴 ISSUE 4: Navigation Goes to Google Maps App (Not In-App)

**Root Cause:**  
`MapScreen.js` called `Linking.openURL()` which opens the external Google Maps app.  
Also the map embed used a dummy/broken API key causing a blank iframe.

**Fix Applied in:** `src/screens/MapScreen.js`

- **No more `Linking.openURL()`** — navigation stays fully in-app
- Replaced broken Google Maps Embed (requires paid key) with **Leaflet.js + ESRI satellite tiles** — zero API key required
- Full holographic Earth view with satellite imagery, scan lines, and corner reticles
- Destinations are geocoded via OpenStreetMap Nominatim (free) and the map flies to them with animation
- Directions generated by AVANT AI brain (no SerpAPI key needed)
- Added `global.__avantNavigate(dest)` bridge so voice commands from HomeScreen navigate the map

---

## 🔴 ISSUE 5: Floating Overlay Disappears When App Closes

**Root Cause:**  
`App.js` never called `startFloatingOverlay()`. The native `AvantModule.kt` didn't exist in the repo.

**Fix Applied in:** `android/app/src/main/java/com/avant/AvantModule.kt`, `App.js`

- `AvantModule.kt` now creates a **draggable floating ⚡ badge** using `SYSTEM_ALERT_WINDOW`
- The badge changes icon: ⚡ (idle) → 🎙 (listening) → ⚙ (thinking) → 💬 (speaking)
- `AvantForegroundService` uses `START_STICKY` so Android restarts it if killed
- `AvantBootReceiver` restarts the service on device reboot
- `App.js` now calls both `startVoiceKernel()` AND `startFloatingOverlay()`

---

## 🔴 ISSUE 6: No Voice Activation — Everything Needs a Button Tap

**Root Cause:**  
All screens required button taps to activate. The voice engine had no continuous mode.

**Fix Applied in:** All screens + voiceEngine.ts

- **HomeScreen**: Orb tap OR wake word starts session — no more Alert.prompt
- **MapScreen**: Voice button + auto-navigates when destination passed from voice command
- **ARScreen**: Voice button triggers VoiceEngine session
- **Navigation**: Say `"open map"`, `"open vision"`, `"navigate to [place]"` to switch screens

## 📋 Deployment Steps

### 1. Replace Files
Copy all files from this fix package into your repo:
```
voice/voiceEngine.ts          → replace existing
src/screens/HomeScreen.js     → replace existing  
src/screens/MapScreen.js      → replace existing
src/screens/ARScreen.js       → replace existing
App.js                        → replace existing
app.json                      → replace existing
src/modules/config.js         → replace existing
android/app/src/main/java/com/avant/AvantModule.kt  → CREATE this file
```

### 2. Update AndroidManifest.xml
Add all permissions and service declarations from:
`android/app/src/main/AndroidManifest_additions.xml`

### 3. Grant Permissions on Device
After installing the APK, go to:
- **Settings → Apps → AVANT → Permissions** → Allow ALL
- **Settings → Apps → AVANT → Display over other apps** → Allow (for floating overlay)

### 4. Build New APK
```bash
eas build --platform android --profile preview
```

### 5. Test Wake Word
1. Open AVANT, let it boot and greet you
2. Lock the screen or switch to another app
3. Say **"Hey Avant"** or **"Whats up Avant"**
4. You should hear **"Yes?"** followed by the listening tone
5. Ask anything — AVANT will speak the answer back
